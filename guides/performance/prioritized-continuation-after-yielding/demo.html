Prioritized Continuation vs Queue Jumping

This demo compares yielding with setTimeout(..., 0) and scheduler.yield(). scheduler.yield() prioritizes the task's continuation over starting other tasks, preventing "Queue Jumping" by background or 3rd party scripts.

Simulating Queue Jumping vs Prioritized Continuation:

  1. Click 1. Schedule "Queue Jumper" Task (Arms the jumper task to compete during the next run).
  2. Click 2a. Run with setTimeout().
    👉 Check Log: The Jumper runs between Task Parts (Interleaving/Queue Jumping).
  3. Click Clear Log.
  4. Click 1. Schedule "Queue Jumper" Task again.
  5. Click 2b. Run with scheduler.yield().
    👉 Check Log: Main task completes fully before Jumper runs (Prioritization).
