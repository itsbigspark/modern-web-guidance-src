Speculative Preloading & Prerendering

Accelerate navigation by fetching and prerendering future content before the user navigates.

Speculation Rules API

The modern way to handle this is the Speculation Rules API.

html
<script type="speculationrules">
{
  "prerender": [
    {
      "source": "document",
      "where": {
        "and": [
          { "href_matches": "/*" },
          { "not": { "href_matches": ["/logout", "/add-to-cart", "/checkout/*"] }}
        ]
      },
      "eagerness": "moderate"
    }
  ]
}
</script>
Best Practices
DO reserve eager prefetches and prerenders ("eagerness" values of "immediate" or "eager") for only the highest-confidence speculative navigations
DO NOT prerender URLs that trigger state changes, like /logout
Fallback strategies

Baseline status: Limited availability

No fallback strategies are required; this feature is safe to use in all browsers. Unsupported browsers will gracefully ignore the <script type="speculationrules"> tag and navigate to the page as normal.

To ensure navigations are as instant as possible, I've implemented the Speculation Rules API in your 
index.html
. This browser feature allows the browser to prerender pages in the background when a user hovers over a link, resulting in near-instant page loads.

I configured it with the following rules:

Scope: It targets all links on the site (/*).
Safety: I explicitly excluded the /logout path to prevent accidental logouts.
Eagerness: I set it to "moderate", which triggers the prerender on hover (or pointer down), balancing performance with resource usage.

Here is the applied configuration:

html
<script type="speculationrules">
  {
    "prerender": [
      {
        "source": "document",
        "where": {
          "and": [
            { "href_matches": "/*" },
            { "not": { "href_matches": "/logout" } }
          ]
        },
        "eagerness": "moderate"
      }
    ]
  }
</script>
